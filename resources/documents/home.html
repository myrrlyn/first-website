This is primarily created as a continuous learning exercise for me to play with various web technologies, including:

  • The Big Three of web languages,

    1. HTML
    2. CSS
    3. JS

  • server-side programming, in languages such as C#, PHP, Ruby, Python, and more JavaScript,

  • good web practices, such as device scaling (translation: not looking like shit on your phone), AJAX transfers, session tracking, and all the other fun behind-the-scenes tricks that make web traffic go smoothly,

  • and lastly, server programming and use in various environments. At completely sporadic intervals, this site will be switching among:

    • GitHub Pages reading static pages out of a repository
    • Amazon AWS — Windows Server 2012 hosting IIS
    • Amazon AWS — Ubuntu hosting Apache
    • Microsoft Azure
    • My personal computer here at Trine
    • The Raspberry Pi hanging off my bed.

    as I learn how to work with the various services, including doing things myself

One of the problems of trying to learn how to do content delivery is not having any content to deliver. Peruse my TES writings, a half-baked coding project that I mostly do to demonstrate some level of competence in new languages, or anything else I happen to have in the nav bar.

Now that more pages are being added, you may notice that there is no actual navigation – the address of your browser never changes and the history stack receives no further data. This is because the main content is swapped in and out using jQuery's AJAX handlers. The site is both static and shallow; everything can be reached from the navbar below the banner image. Since the transmitted documents are very small and contain no embedded CSS or JavaScript, there should not be any delay, but even if there is it is certainly less than requesting an entirely new page which will have the framework duplicated.

Ajax is cool.

Sorry if you get a bit of unresponsiveness, though.
